# API

## Overall Picture

An API is a way for one system to ask another system for something in a clear, repeatable format.

## Distinctive Features

- It defines what can be requested.
- It defines how requests and responses are shaped.
- It reduces guesswork between systems.

## Internal Logic

A client sends a request, the server interprets it, and the server returns a response under agreed rules.

## Top 10 Knowledge Points

1. API means Application Programming Interface.
2. It is a rule-based connection surface.
3. It can be local or remote.
4. Web APIs often use HTTP.
5. Requests usually contain method, path, headers, and body.
6. Responses usually contain status, headers, and body.
7. Good APIs are predictable.
8. Good APIs are documented.
9. Errors are part of the contract.
10. APIs are easier to use when examples are included.
