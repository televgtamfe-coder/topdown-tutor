# Source Acquisition

## Non-Negotiable Source Rule

Easy-to-understand explanation does not mean child-grade source material.

- Simplify the explanation, not the authority level.
- Prefer adult-level, authoritative, or otherwise topic-appropriate materials.
- Use children's or elementary-school materials only if the user explicitly asks for that source level.

## Topic Routing

- Library, framework, API, or tool:
  - official docs first
  - Context7 when available
  - official examples
  - primary specs when relevant
- Broad concept:
  - textbooks
  - review articles
  - reputable educational material
  - encyclopedic overviews
- Historical, political, intellectual, or ideological topic:
  - official or primary source material when relevant
  - academic or reference overviews
  - reputable encyclopedic sources
  - explicit source-perspective labeling when interpretation may differ
  - a combined chain of overview + original text + decomposition
- Applied skill:
  - worked examples
  - practice tasks
  - implementation guides
- Project-oriented topic:
  - reference repositories
  - architecture writeups
  - examples with code

## Output Requirement

Convert gathered sources into a compact learning pack instead of sending a raw list.

## Three-Part Chain For Canonical-Text Topics

When a topic has important original texts, keep both routes:

1. Overview
   - choose 1 to 3 orientation sources
2. Original text
   - choose the canonical texts or primary-source cluster
3. Decomposition
   - break each text into context, core claim, plain-language meaning, and why it matters

Do not force the learner to choose between summary material and direct source material when both are useful.
