# 从0到通

[English README](./README.md)

把难知识从上到下讲清楚，把学习材料一起找好，再检查学习者是不是真的懂了。

从0到通先用通俗易懂的方式建立理解，再逐层深入。它保留材料权威性，支持“综述 + 原典 + 拆解”，并用少量高价值问题做理解检查，而不是只堆信息。

## 它是什么

从0到通是一个可复用的学习型 skill 包。

它把四件事放进同一条流程里：

1. 把一个主题整理成可讲的知识地图
2. 先把材料找好，不让学习者自己到处搜
3. 先讲懂，再逐步讲深
4. 用少量关键问题检查是否真的理解

它适合这些很容易被讲乱的主题：

- API、React Hooks 这类技术主题
- 概率这类概念性主题
- 《孙子兵法》这类原典主题
- 需要既看综述、又接触原始材料的理论型主题

## 它为什么不一样

很多“讲解型”提示词只做到“解释一下”。从0到通更严格。

- 它有固定讲解顺序：整体图景、特色、内部逻辑、10 个最重要知识点。
- 它先用通俗易懂的方式讲，但不会因为要讲得易懂，就自动降低材料权威性。
- 它保留“综述 + 原典 + 拆解”三段式路径。
- 它每轮最多只抓 3 个关键检查点，不会把问题问散。
- 它要求每个新知识点都带一个简单例子。
- 它要求专业词或英文第一次出现时，给出原词、中文翻译、通俗中文解释。

## 它怎么教

它的固定节奏是：

1. 起点不足时先做零基础总览拆解
2. 先讲整体图景
3. 再讲特色
4. 再讲内部逻辑
5. 最后讲 10 个最重要知识点
6. 再做关键检查

每个主题分三层解释：

1. 通俗易懂的理解层
2. 准确的日常表达层
3. 更专业的深入层

每个新知识点都配这些元素：

- 一个简单例子
- 如果出现专业词或英文，第一次出现时做术语翻译说明
- 如果知道学习者的职业或环境，就补一个相关应用

## 它怎么获取材料

从0到通不会把所有主题都用同一种材料路线处理。

- 库、框架、API、工具类主题，优先官方文档、官方示例、第一手资料。
- 概念类主题，优先教材、综述、靠谱的教育材料。
- 历史、思想、原典类主题，优先综述材料、原始文本、拆解说明一起用。
- 应用类主题，优先示例、练习、实操材料。

最核心的一条规则是：

**让解释更易懂，不等于降低材料的权威性。除非学习者明确要求，否则不自动换成低年级材料。**

## 原典与拆解

当一个主题有重要原典时，从0到通默认保留三段：

1. 综述
2. 原典
3. 拆解

也就是先让学习者知道大图景，再接触原文，再把原文拆成可吸收的单元：

- 历史背景
- 核心主张
- 通俗含义
- 重要术语
- 在整体体系里的位置
- 常见误解

这样可以同时避免两个常见问题：

- 只停留在别人替你总结的层面
- 把原文直接扔给学习者，没人帮忙拆

## 安装

推荐使用 GitHub CLI `gh 2.90.0+` 安装。

给 Codex 安装到用户级目录：

```bash
gh skill install televgtamfe-coder/topdown-tutor knowledge-coach --agent codex --scope user
```

安装前先预览：

```bash
gh skill preview televgtamfe-coder/topdown-tutor knowledge-coach
```

从本地仓库安装：

```bash
gh skill install D:\topdown-tutor knowledge-coach --from-local --dir C:\Users\MeetYou\.codex\skills --force
```

可选的 Codex plugin 打包目录现在位于 `plugins/topdown-tutor`。

## 使用方式

你可以直接这样提：

```text
从上到下给我讲 API 是什么，把合适材料一起找好，再检查我是不是真的懂了。
```

```text
从上到下给我讲 React Hooks。要通俗易懂，但不要自动降低材料权威性。
```

```text
从上到下给我讲概率，用例子和关键检查点帮我真正理解。
```

```text
把《孙子兵法》按原典主题来讲，用“综述 + 原典 + 拆解”的方式带我学。
```

## 输出格式

标准输出结构是：

1. 一句话定义主题
2. 整体图景
3. 特色
4. 内部逻辑
5. 10 个最重要知识点
6. 每个新知识点一个简单例子
7. 术语首次出现时的翻译和通俗解释
8. 最多 3 个关键检查点

如果是原典型主题，还会多出：

1. 综述材料
2. 原典材料
3. 拆解路径

## 常见问题

### 通俗易懂是不是等于儿童材料？

不是。它指的是讲法更易懂，不是把材料自动降级。

### 这是不是只能讲技术主题？

不是。它也适合概念型、理论型和原典型主题。

### 它会不会一次问很多问题？

不会。每轮最多 3 个关键检查点。

### 它支持原典拆解吗？

支持。这是它最核心的设计之一。
