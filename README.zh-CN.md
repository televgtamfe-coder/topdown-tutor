把难知识讲成真正能懂、能复述、能继续学的一次完整学习回合。

从0到通不是“解释一下”。它把材料获取、知识拆解、讲解节奏、理解检查放进同一条流程里，让一个主题从“看不懂”，走到“能说清”，再走到“能继续往下学”。

如果你要找的不是一个只会堆信息的提示词，而是一套能长期复用的学习方法，这个项目就是为这件事设计的。

## 30 秒看懂

- 先用通俗易懂的方式建立理解，再逐层深入。
- 先讲整体图景，再讲特色，再讲内部逻辑，最后压成 10 个最重要知识点。
- 不会因为要讲得易懂，就自动把材料降成低权威版本。
- 原典型主题默认保留“综述 + 原典 + 拆解”三段式路径。
- 每轮最多只抓 3 个关键检查点，不把问题问散。
- 每个新知识点都配一个简单例子。
- 专业词或英文第一次出现时，补原词、中文翻译、通俗中文解释。

## 为什么它容易被收藏和转发

- 它解决的不是“不会解释”，而是“不会真正教会”。
- 它把“找材料”这件最容易卡住学习者的事一起解决了。
- 它既能讲 API、React Hooks，也能讲概念、原典、思想、理论。
- 它的输出结构稳定，很适合沉淀成 skill、agent、prompt 或团队内部教学标准。

## 快速开始

推荐使用 GitHub CLI `gh 2.90.0+` 安装。

给 Codex 安装到用户级目录：

```bash
gh skill install televgtamfe-coder/topdown-tutor knowledge-coach --agent codex --scope user
```

安装前先预览：

```bash
gh skill preview televgtamfe-coder/topdown-tutor knowledge-coach
```

从本地仓库安装：

```bash
gh skill install D:\topdown-tutor knowledge-coach --from-local --dir C:\Users\MeetYou\.codex\skills --force
```

可选的 Codex plugin 打包目录位于 `plugins/topdown-tutor`。

## 一次学习回合里，你会得到什么

- 一句话定义主题
- 起点不足时的零基础总览拆解
- 先给整体图景
- 再讲这个主题最有辨识度的特色
- 再讲内部逻辑或运行逻辑
- 最后压成 10 个最重要知识点
- 每个新点都给一个简单例子
- 术语首次出现时给翻译和通俗说明
- 最多 3 个关键检查点
- 如果知道你的职业或环境，再补一个贴近场景的应用建议

## 它怎么讲

固定讲解节奏是：

1. 起点不足时先做零基础总览拆解
2. 先讲整体图景
3. 再讲特色
4. 再讲内部逻辑
5. 最后讲 10 个最重要知识点
6. 再做关键检查

每个主题分三层解释：

1. 通俗易懂的理解层
2. 准确的日常表达层
3. 更专业的深入层

## 它怎么找材料

从0到通不会把所有主题都用同一种材料路线处理。

- 库、框架、API、工具类主题，优先官方文档、官方示例、第一手资料。
- 概念类主题，优先教材、综述、靠谱的教育材料。
- 历史、思想、原典类主题，优先综述材料、原始文本、拆解说明一起用。
- 应用类主题，优先示例、练习、实操材料。

最核心的一条规则是：

**让解释更易懂，不等于降低材料的权威性。除非学习者明确要求，否则不自动换成低年级材料。**

## 原典主题模式

当一个主题有重要原典时，从0到通默认保留三段：

1. 综述
2. 原典
3. 拆解

也就是先让学习者知道大图景，再接触原文，再把原文拆成可吸收的单元：

- 历史背景
- 核心主张
- 通俗含义
- 重要术语
- 在整体体系里的位置
- 常见误解

这样可以同时避免两个常见问题：

- 只停留在别人替你总结的层面
- 把原文直接扔给学习者，没人帮忙拆

```

```

```text

```

## 适合哪些主题

这套方法特别适合这些容易被讲乱、讲浅、或者讲得太硬的主题：

- API、React Hooks 这类技术主题
- 概率这类概念性主题
- 《孙子兵法》这类原典主题
- 需要既看综述、又接触原始材料的理论型主题

## 常见问题

### “通俗易懂”是不是等于儿童材料？

不是。它指的是讲法更易懂，不是把材料自动降级。

### 它会不会一次问很多问题？

不会。每轮最多 3 个关键检查点。

### 它支持原典拆解吗？

支持。这是它最核心的设计之一。

### 它是不是只能讲技术主题？

不是。它也适合概念型、理论型、历史型和原典型主题。

## 案例：真实对话节选（毛泽东思想）

下面这段保留原始互动节奏，不做二次润色，目的是展示这套方法怎么把“泛泛而谈”一步步压到“自己能说清”。

> Assistant：我先按你是想问“毛泽东思想”来讲。
>
> Assistant：如果把它用最通俗的方式讲，它就是：把马克思列宁主义的基本原理，放到中国当时的实际情况里去用，不是照抄，而是“按中国的地，种中国的庄稼”。
>
> Assistant：你先试着用一句话说说看：毛泽东思想最核心的一点，你现在觉得是什么？
>
> User：马列主义在中国当时情况下的应用，是因为是一套思想体系。
>
> Assistant：很接近。我帮你压得更准一点：毛泽东思想，是把马克思列宁主义同中国具体实际结合起来形成的一套思想体系。
>
> Assistant：如果把它再压缩成 4 个字，你觉得最重要的方法味道像什么？
>
> User：实事求是。
>
> Assistant：对。那你再用自己的话说说，什么叫“实事求是”？
>
> User：从实际出发 靠研究弄清情况再决定方针。
